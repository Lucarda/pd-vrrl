Port-forwarding:

There are cases where you have to open your vrrl UDP port in your router. Thare are also cases where you can not do that but you can use a VPN.

Some links that covers port-forwarding:

https://www.noip.com/support/knowledgebase/general-port-forwarding-guide/

VPN:

Hamachi

brief: at least one of the members of the group has to sign-up for an account (the free account is for 5 machines on the network) on https://www.vpn.net and create a "mesh" network. All members have to install a client to join that network.

Any of the members (only one) must create an [aoo_server 10102] object. This object could have had any other port number in the range 1025-65000 instead of 10102.

Then all members (including the one that created the [aoo_server 10102]) must connect to that server using the "hamachi/ip" "port number" of the machine that has created the [aoo_server 10102]

note: hamachi ip's normally start 25.x.x.x




OPENVPN

If someone in the group has a linux machine it should be easy to setup a VPN using this script:

https://github.com/angristan/openvpn-install

that sets an OPENVPN server that can be used by the ones that cant do port-forwarding or by all members of the group.
